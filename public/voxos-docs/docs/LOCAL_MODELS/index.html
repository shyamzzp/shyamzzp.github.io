Local Model Manager

Goal​

Let VOXOS keep a local model catalog for offline voice/LLM work with a native GGUF runtime available on iPhone.

Supported Model Files​

The current picker accepts:

  • .gguf
  • .mlmodel
  • .mlmodelc
  • .mlpackage

Users can import files from the iOS Files picker or download a model from an HTTP/HTTPS URL in Settings.

VOXOS also ships with a default bundled GGUF model:

  • voice_xcode/VOXOS/BundledLocalModels/SmolLM2-135M-Instruct-Q4_K_M.gguf

On launch, the app registers this bundled model in SQLite and selects it as the local LLM when the saved selection is missing, invalid, or None. This means voice replies have a packaged local fallback without requiring the user to import or download a model first. User-selected valid local models are preserved.

Settings also includes starter GGUF presets:

  • Qwen 0.5B for a balanced first download.
  • SmolLM2 135M for the smallest quick test.
  • TinyLlama 1.1B for a larger local model.

The custom URL field remains available for any user-provided GGUF/Core ML model URL.

Capability Labels​

Settings labels each local model and preset with conservative best-use tags:

  • Short replies for compact assistant answers.
  • Intent JSON for structured command output.
  • Command routing for opening apps and routing actions.
  • Summarization for condensing local context.
  • OCR cleanup for normalizing receipt or document text.
  • Persona extraction for proposing user facts before approval.
  • Custom local for imported models that should be validated before sensitive workflows.

Unknown GGUF/Core ML files keep safe defaults until the runtime test and manual usage prove they are a better fit.

Local Storage​

Model files are copied into:

  • Application Support/VOXOS/Models/

Bundled model files stay read-only inside the app bundle and are referenced by path in SQLite. If the app is reinstalled and the bundle path changes, VOXOS refreshes the SQLite row on the next launch.

Model metadata is stored in SQLite:

  • local_models.id
  • local_models.name
  • local_models.kind
  • local_models.path
  • local_models.size
  • local_models.source
  • local_models.imported_at

Runtime Status​

This implementation adds the storage, settings, downloader, importer, selector, deletion flow, runtime readiness test, and an iOS-only llama.xcframework linked into the native Xcode target.

The Settings card has a local model test button that reports:

  • Missing or unselected model.
  • GGUF file present and loadable through llama.cpp.
  • Core ML model compile/load success or the Core ML load error.

Native text generation status:

  • GGUF uses voice_xcode/VOXOS/LlamaRuntime.swift, a compact Swift wrapper around llama.cpp. It loads the selected GGUF model, keeps one active context cached by model path, and generates short voice replies.
  • Core ML needs a concrete compiled model adapter and prompt/token interface.
  • Speech recognition can prefer Apple's on-device recognizer when local LLM mode is enabled.

For best phone-side performance, start with the SmolLM2 135M or Qwen 0.5B Q4 preset. Larger GGUF files can load slowly, increase app memory pressure, or fail on older devices.
