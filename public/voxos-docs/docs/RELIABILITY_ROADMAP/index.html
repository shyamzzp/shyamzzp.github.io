VOXOS Reliability Roadmap

Direction​

VOXOS is moving toward a local-first personal data OS. The product should stay useful only if the core flows are reliable: voice routing, permission setup, local indexing, SQLite persistence, provider integrations, Mac Link control, and background media.

Current Reliability Pass​

  • Add visible System Health in Settings so setup gaps are discoverable before testing.
  • Keep setup state local and derived from existing SQLite/settings state instead of adding new storage.
  • Keep manual test steps in docs/TESTING.md so every new feature can be checked against navigation, voice, picker, provider, and background-audio behavior.
  • Add SQLite schema version tracking and migration visibility so installed devices can upgrade local data in place without rebuilding tables.
  • Create timestamped SQLite backups before versioned migrations on existing installs, while keeping OAuth/API/card secrets in iOS Keychain.
  • Add stable accessibility identifiers and XCUITest coverage for Settings health/storage, Terminal detail setup guidance, and dashboard detail navigation.
  • Add actionable Terminal error guidance for disabled Mac Link, missing Mac Link fields, disabled terminal bridge, tmux setup, LAN reachability, and macOS Automation/Accessibility permission failures.

Next Gaps To Close​

  • Add a formal migration test harness with seeded old databases and expected post-migration row counts.
  • Add a unified setup router so every failed voice/provider/action opens the exact Settings subsection or app detail page that can fix it.
  • Add repeatable XCUITests for detail navigation, Settings health visibility, voice button failure UI, Files picker entry, and Music detail launch.
  • Add provider-specific contract tests for Composio, Google, Hugging Face, and Mac Link clients using fixture responses.
  • Add a privacy review screen listing what is local SQLite, what is Keychain, what is App Group widget state, and what leaves the device.
  • Add a release checklist that verifies background audio, widgets, Live Activities, notifications, and Face ID wallet flows on a physical iPhone.

Manual Device Checks​

  • Lock Screen audio controls cannot be fully verified from terminal automation; test on the connected iPhone by playing a VOXOS Music track and locking the phone.
  • iOS Files picker selection also needs manual confirmation because terminal automation cannot select a real user folder inside the system picker.
  • OAuth account consent and provider dashboards need manual confirmation whenever scopes or redirect settings change.
