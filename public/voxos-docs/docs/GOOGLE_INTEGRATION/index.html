Google Integration

VOXOS now has a native Google OAuth foundation for profile/account setup and voice-triggered access requests.

Entry Points​

  • Profile/account button on the main dashboard.
  • Account button on the Persona screen.
  • Settings > Google.
  • Google dashboard grid item after at least one account is connected.
  • Voice commands such as can you access Gmail, connect Google, or sign in to Gmail.
  • Multiple Google accounts can be connected. VOXOS keeps one active Google account for voice routing and future sync work.

Current Scope​

The default requested scopes are:

  • openid
  • email
  • profile
  • https://www.googleapis.com/auth/gmail.readonly
  • https://www.googleapis.com/auth/drive.metadata.readonly
  • https://www.googleapis.com/auth/photoslibrary.readonly
  • https://www.googleapis.com/auth/calendar.readonly
  • https://www.googleapis.com/auth/contacts.readonly

The service scopes are read-only where Google supports read-only access. Indexing and summarization should still be added as a follow-up after the account token is available.

In-App Setup Guidance​

Account Integrations now shows provider-specific setup steps. Google explains how to create/select a Google Cloud project, configure the OAuth consent screen, create the iOS OAuth client for com.shyamzzp.voxos, copy the iOS client ID, copy the reversed client ID URL scheme, and request read-only scopes for Gmail, Drive metadata, Google Photos, Calendar, and Contacts.

Required Google Setup​

Create an iOS OAuth client in Google Cloud for the VOXOS bundle ID:

  • com.shyamzzp.voxos

Then provide:

  • Google iOS OAuth client ID.
  • Reversed client ID URL scheme.

The reversed client ID must also be present in the iOS app's CFBundleURLTypes. Runtime settings can save the values locally, but iOS URL schemes are app-bundle metadata, so a real production setup should add the final reversed scheme to voice_xcode/VOXOS/Info.plist and rebuild.

Runtime Behavior​

When a user asks for Gmail/Google access by voice, VOXOS checks the local Google connection:

  • If connected, VOXOS opens the relevant Google service detail page or the Google parent card and reports the active connected account.
  • If not connected, VOXOS opens Account Integrations and attempts Google sign-in.
  • If the OAuth client ID or reversed URL scheme is missing, VOXOS keeps the sheet open and reports the missing setup.
  • Users can add another Google account, choose the active account, or disconnect one account without removing the others.
  • The parent Google grid item shows the active account email. Tapping it opens a services sheet for the connected account(s).
  • The services sheet shows every supported Google service and marks each service as Granted or Reconnect depending on the scopes returned by Google.
  • Gmail, Drive, Photos, Calendar, and Contacts detail pages currently show account/scope state and the intended metadata surfaces. Live provider API fetch/index work remains the next step.

Existing connected Google accounts must reconnect after new scopes are added. Google does not silently grant new scopes to an old token.

Security Notes​

  • Token storage: Google OAuth tokens are stored in Keychain-backed account rows; granted scopes and active account metadata stay in local SQLite.
  • Outbound data: VOXos requests only metadata/content allowed by the granted Google scopes for the selected account.
  • Destructive actions: current Google surfaces are read/setup-first; future send, edit, delete, or move actions must be explicitly mapped.
  • Confirmation: scope expansion requires reconnect, and any future write action must show an approval card before provider handoff.
  • Revoke path: disconnect the account in VOXos and revoke VOXos access from Google Account > Security > Third-party access.

Storage​

Google config persists in SQLite settings:

  • google_client_id
  • google_redirect_scheme
  • google_scopes
  • google_active_account_id

Google account credentials persist in the SQLite google_accounts table:

  • id
  • email
  • access_token
  • refresh_token
  • id_token
  • granted_scopes
  • token_expiry
  • connected_at
  • updated_at

Older single-account prototype keys are migrated into google_accounts on launch when present.

Move token storage to Keychain before broad testing or release.
