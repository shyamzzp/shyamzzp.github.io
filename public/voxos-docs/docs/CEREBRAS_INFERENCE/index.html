Cerebras Inference

Goal​

Use Cerebras as a cloud LLM inference option while local GGUF/Core ML runtime work continues.

App Settings​

In the Settings card:

  • Set engine to Cloud.
  • Set cloud provider to Cerebras.
  • Choose a Cerebras model.
  • Enter a Cerebras API key.
  • Tap Test Cerebras to verify the selected key/model can reach the API.
  • Tap Save settings.

The app stores the selected provider, model, and API key in local SQLite with the rest of the voice settings.

Current Models​

The first model options are:

  • llama3.1-8b
  • gpt-oss-120b
  • qwen-3-235b-a22b-instruct-2507

Capability Labels​

Settings shows best-use labels beside cloud model choices:

  • llama3.1-8b: short replies, intent JSON, command routing, and summarization.
  • gpt-oss-120b: cloud reasoning, summarization, provider chat, OCR cleanup, and Persona extraction.
  • qwen-3-235b-a22b-instruct-2507: intent JSON, command routing, provider chat, summarization, and Persona extraction.

OpenRouter model IDs are labeled from the configured model name with conservative defaults. Larger GPT/Claude/Gemini/Qwen-like IDs are treated as broader reasoning models; Llama/Mistral-like IDs are treated as compact command and summarization models.

Request Shape​

VOXOS calls:

  • POST https://api.cerebras.ai/v1/chat/completions

with an OpenAI-compatible chat payload and Authorization: Bearer <CEREBRAS_API_KEY>.

The test connection sends a tiny prompt that expects VOXOS_OK back. A successful test displays Cerebras connected.

Notes​

This is direct from the iOS app for now. For a production website, the same request shape should usually be proxied through a backend so the browser never receives the Cerebras API key.
