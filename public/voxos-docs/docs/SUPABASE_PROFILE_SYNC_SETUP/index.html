Supabase Profile Sync Setup

VOXos uses Supabase for active-profile Mac Link endpoint sharing. Table names must keep the voxos_ prefix because this Supabase project can host other apps.

Current Project​

  • Project ref: lpehkeyhpwxljdylbhkc
  • Project URL: https://lpehkeyhpwxljdylbhkc.supabase.co
  • Table: public.voxos_profile_maclink_endpoints
  • Migration: supabase/migrations/20260508141347_voxos_profile_sync.sql

Database​

The remote migration has been applied. To verify migration history later:

supabase migration list

Use the repo verifier for the table and Auth provider checks:

scripts/check_supabase_profile_sync.sh

It exits nonzero until Google is enabled in Supabase Auth.

Remaining Auth Setup​

Supabase Auth still needs the Google external provider enabled before VOXos can exchange the active Google ID token for a Supabase session.

Create or use a Google OAuth Web application client and add this authorized redirect URI:

https://lpehkeyhpwxljdylbhkc.supabase.co/auth/v1/callback

Then enable Google in Supabase Auth with:

  • Google Web client ID
  • Google Web client secret

Do not commit the client secret.

You can enable the provider from this repo when the Supabase Management API token and Google Web client credentials are available in your shell:

export SUPABASE_ACCESS_TOKEN=...
export SUPABASE_AUTH_EXTERNAL_GOOGLE_CLIENT_ID=...
export SUPABASE_AUTH_EXTERNAL_GOOGLE_CLIENT_SECRET=...
scripts/enable_supabase_google_provider.sh

The script only patches the Google Auth provider settings for project lpehkeyhpwxljdylbhkc, then runs scripts/check_supabase_profile_sync.sh.

VOXos Settings > Supabase has a Check Supabase button. It checks the Mac Link table through PostgREST and confirms whether Supabase Auth reports Google as enabled.

After enabling it, verify from the repo:

scripts/check_supabase_profile_sync.sh

The script should print Google Auth provider: true and Supabase profile sync is ready.
