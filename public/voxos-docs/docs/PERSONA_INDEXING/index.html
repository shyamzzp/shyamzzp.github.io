Persona Indexing Direction

Goal​

VOXOS should become a local-first personal context layer. The app should index approved local data and gradually build a Persona knowledge base that improves search, summaries, reminders, and voice answers over time.

Current First Step​

The native app now includes a Persona dashboard card and a fixed bottom Persona action. Persona currently supports:

  • Typed knowledge entry.
  • Voice entry through commands such as remember that I am senior software engineer at Oracle.
  • File upload through the iOS Files picker for text-like files.
  • Local fact extraction for current role, current company, current work, location, skills, interests, and education.
  • Fact rows with source text category and confidence.
  • Approval-first review for new or conflicting facts.
  • Conflict review rows show current value, proposed value, source, confidence, change probability, and suggested merge before the user approves or rejects.
  • Per-fact delete and full clear actions.
  • SQLite persistence in persona_facts.
  • Pending review items persist locally in persona_review_items so they survive relaunch until approved or discarded.

Example input:

hello I am senior software engineer at Oracle

Current extraction:

  • Current role: Senior Software Engineer
  • Current company: Oracle
  • Current work: Senior Software Engineer at Oracle

Privacy Boundary​

iOS apps cannot silently scrape the entire phone. VOXOS should only collect data through explicit user action or system permission:

  • Files selected through the iOS Files picker.
  • Notes created in VOXOS.
  • Message text manually imported by the user.
  • Spoken status check responses captured after the user answers the in-app Call card.
  • Location/weather after iOS permission.
  • Public web sources configured by the user.

Hidden SMS database access, private app data scraping, and whole-device crawling are not available to third-party iOS apps and should not be treated as product assumptions.

Metadata Storage​

Local metadata should stay in Application Support/VOXOS/voxos.sqlite until the schema stabilizes.

Current table:

  • persona_facts: normalized facts with source, confidence, and timestamps.
  • persona_review_items: pending Persona updates with proposed value, existing value, source, confidence, change probability, suggested merge context, and timestamps.

Conflict Rules​

  • Incoming facts are compared by normalized attribute.
  • If the same attribute and same value already exists, VOXOS skips the duplicate proposal.
  • If the same attribute has a materially different value, VOXOS stages it as a conflict instead of silently replacing the current fact.
  • Approving a conflict replaces the older fact with the proposed value.
  • Rejecting a conflict leaves the existing fact unchanged and removes only the pending review row.

Recommended future tables:

  • profile_sources: user-approved sources and provider details.
  • profile_embeddings: vector references for semantic profile retrieval.
  • profile_events: observed changes over time.
  • status_checks: in-app call responses that capture the user's current state.

Large indexes, embeddings, and extracted artifacts should live under Application Support/VOXOS/IndexCache/.

Next Steps​

  • Generate persona facts from indexed files, notes, imported messages, and location context automatically after user consent.
  • Add provenance to every fact so users can inspect where it came from.
  • Add controls to delete a source or clear the full Persona knowledge base.
  • Add optional public web source configuration with explicit consent before crawling.
