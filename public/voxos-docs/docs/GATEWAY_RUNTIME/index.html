Gateway Runtime

VOXos now has a backend Gateway foundation for the OpenClaw-style runtime direction without changing the native iOS UI.

What It Provides​

  • Gateway sessions and message history backed by local SQLite.
  • Channel ingress for VOXos iOS, WebChat, CLI, Telegram, Slack, Discord, WhatsApp-style payloads, and trusted Mac bridges.
  • Route proposals that map requests to VOXos skills, tool profiles, risk level, and confirmation requirements.
  • Node registration for trusted devices such as Mac Link.
  • Built-in skill registry for local search, daily briefing, Persona memory, Composio Gmail/GitHub reads, Mac Link, Terminal Bridge, and receipt scan workflows.
  • Tool profiles for minimal, read-only, personal, Mac control, and developer modes.
  • Background job records for future briefing refreshes, provider sync, file indexing, and channel adapter work.
  • Safe read execution adapters for Gateway local search, Gateway briefing state, GitHub/Composio read paths, and Mac Link status.
  • Optional channel signature verification, token auth, rate limiting, and metadata/payload redaction.

Run​

python3 -m venv .venv
.venv/bin/pip install -r voice_backend/requirements.txt
scripts/start_voxos_gateway.sh

Process pending jobs:

scripts/voxos_gateway_worker.py --once
scripts/voxos_gateway_worker.py --interval 30 --limit 10

The default Gateway database is ignored by git:

.codex/voxos-gateway.sqlite

Useful environment variables:

export VOXOS_GATEWAY_HOST=127.0.0.1
export VOXOS_GATEWAY_PORT=8000
export VOXOS_GATEWAY_TOKEN="shared-token"
export VOXOS_GATEWAY_DB="$PWD/.codex/voxos-gateway.sqlite"
export VOXOS_SESSION_STORE=memory
export VOXOS_GATEWAY_RATE_LIMIT_PER_MINUTE=60

When VOXOS_GATEWAY_TOKEN is set, Gateway endpoints require either:

Authorization: Bearer shared-token

or:

X-VOXOS-Gateway-Token: shared-token

Channel Setup​

Use the helper script to configure or print deployment values:

scripts/configure_gateway_channel.py telegram \
  --gateway-url "https://gateway.example.com" \
  --bot-token "$TELEGRAM_BOT_TOKEN" \
  --secret-token "$VOXOS_TELEGRAM_SECRET_TOKEN"

scripts/configure_gateway_channel.py slack --gateway-url "https://gateway.example.com"
scripts/configure_gateway_channel.py discord --gateway-url "https://gateway.example.com"
scripts/configure_gateway_channel.py whatsapp --gateway-url "https://gateway.example.com"

Optional channel-specific verification:

export VOXOS_TELEGRAM_SECRET_TOKEN="telegram-secret-token"
export VOXOS_SLACK_SIGNING_SECRET="slack-signing-secret"
export VOXOS_DISCORD_PUBLIC_KEY="discord-ed25519-public-key"
export VOXOS_WHATSAPP_VERIFY_TOKEN="whatsapp-verify-token"
export VOXOS_WHATSAPP_APP_SECRET="whatsapp-app-secret"

Mac Link Node Registration​

Mac Link can advertise itself as a Gateway node:

export VOXOS_GATEWAY_URL="http://127.0.0.1:8000"
export VOXOS_GATEWAY_TOKEN="shared-token"
python3 scripts/voxos_mac_link_server.py

It also exposes a node payload without registering:

GET /api/gateway/node

When registered, Mac Link sends periodic heartbeats. Configure the interval with:

export VOXOS_GATEWAY_HEARTBEAT_SECONDS=60

Safety Model​

Gateway routing does not execute destructive actions directly. It stages a route proposal with:

  • skill_id
  • tool_profile_id
  • risk
  • confirmation_required
  • proposed_action
  • args

Mac control, terminal, write, and destructive routes are confirmation-gated so the native VOXos approval model remains the control surface.

POST /gateway/execute only runs safe read adapters directly. Write, terminal, destructive, and Mac-control routes return needs_confirmation unless the caller marks the request as approved, and unsupported executors remain staged.

Read adapter environment:

export GITHUB_TOKEN="github-token-for-direct-github-reads"
export VOXOS_COMPOSIO_API_KEY="composio-key"
export VOXOS_COMPOSIO_USER_ID="voxos-user"
export VOXOS_COMPOSIO_GITHUB_ACCOUNT_ID="connected-account-id"
export VOXOS_COMPOSIO_GMAIL_ACCOUNT_ID="connected-account-id"
export VOXOS_COMPOSIO_BASE_URL="https://backend.composio.dev/api/v3.1"

.venv/bin/python scripts/validate_gateway_runtime.py
