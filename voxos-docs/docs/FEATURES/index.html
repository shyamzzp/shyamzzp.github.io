VOXOS Feature Log

This folder records product features and implementation notes for the native Xcode app.

Current Features​

  • Compact icon-first dashboard with an in-app VOXos wordmark matching the installed app icon style, installed app icon assets, and logo options in docs/logo-options/.
  • Unified command routing now records a local SQLite audit trail with source, heard text, interpreted intent, action, target, provider, mode, risk, result, and timestamp. Voice/text commands can open apps, run global search, stage Persona updates, connect providers, refresh files, and toggle pins through one path.
  • Voice detail shows a compact recent command history section so users can inspect the latest routed commands without leaving the chat surface.
  • VOXos now attempts a structured LLM intent pass before keyword routing. The configured local GGUF model or selected cloud provider is asked for strict JSON containing action, target, provider, payload, query, risk level, confirmation requirement, confidence, and reason. Valid safe intents execute through the existing local router, risky intents become Confirm/Cancel cards, and invalid or low-confidence LLM output falls back to deterministic routing.
  • Casual Voice detail chat such as greetings now bypasses the structured command-classification pass, clears stale spoken output before each request, and ignores duplicate submits while one chat request is already processing. VoxChat now shows an explicit answer-vs-grid-action classification step, only routes dashboard apps from exact aliases or navigation phrases, and keeps normal questions that merely mention an app in chat response mode. A lightweight local semantic router, inspired by OSS semantic-routing patterns, now assigns route IDs, confidence, and reasons before executing supported grid actions or falling through to chat.
  • Voice detail now has a compact processing row with a Stop action, duplicate-send status feedback, route metadata for Mode/Action/Target/Provider/Risk, and quick command suggestion chips for common commands such as opening Composio, searching files, adding a note, and refreshing files.
  • Voice capture now guards the footer finalization path so one long-press release can only submit once, suppresses identical repeated transcripts for a short window, and handles simple greetings such as hello locally with one short reply instead of invoking the LLM pipeline.
  • Voice chat now sanitizes every assistant response before display, stripping echoed User:/VOXos: prompt turns, suppressing repeated copies of the user's text, clipping long replies, and replacing echo-heavy model output with one concise fallback. The Voice detail page also has its own hold-to-speak mic button next to the chat input.
  • Voice chat now shows a command router status card with the raw heard text, the interpreted intent VOXos will use, and Confirm/Cancel controls before risky actions run. File reindexing, note creation, Persona staging, pin changes, Mac Link tests, Google OAuth, and Composio tests now require approval instead of executing immediately from a typed or spoken command.
  • Provider-aware command routing can open connected Composio services such as GitHub, Gmail, Drive, Calendar, Slack, Notion, or Photos when those accounts are available. If a native Google service has scope access, VOXos routes to the matching Google detail page; otherwise it opens Composio setup guidance for that service.
  • Search, Briefing, and Providers are first-class dashboard apps. Search scans local files, notes, imported messages, Persona facts, schedule events, travel metadata, VOXos notifications, Composio history, and command history. Every global search result shows compact provenance and an expandable source panel with provider, source table, indexed timestamp, permission scope, and freshness/stale/cache status. Briefing composes local context, next schedule, travel watch, notification state, Persona approvals, status calls, setup gaps, and file-index readiness. Providers shows local database, voice, local LLM, Google, Composio, Hugging Face, Mac Link, Terminal, Files, Music, Photos, Schedule, Travel, Receipts, Persona, Wallet Security, and widget readiness with a compact ready/setup/off summary, blocker guidance, and quick refresh actions.
  • Read Later is a native grid micro-app for saved links, articles, PDFs, videos, summaries, tags, notes, and follow-up questions. It stores local metadata in SQLite, adds saved items to global search with provenance, supports manual capture/search/archive/delete, and exposes voice commands for save, search, summarize unread items, archive, and open.
  • Daily Briefing items show source chips, a refresh action for local context and notification state, and an Explain action that asks the configured local LLM for one compact meaning summary with a deterministic fallback when the model is unavailable. Briefing rows are ranked with an explainable priority score using base urgency, user pins, unresolved actions, source reliability, and recency; each row shows the compact score and reason chips.
  • Files indexing stores knowledge-base metadata for user-selected Files sources, including source type, MIME/UTType, tags, extracted text, creation date when iOS exposes it, permission context, and stale/current state. Reindexing marks older rows stale before fresh rows are upserted, and Files can export a compact metadata preview or delete individual indexed rows. VoxChat can run local asset tasks such as get me photos from the last week, returning Search results across indexed image files, cached Google Photos metadata, and saved receipt scans with provenance. Recent image file lookup uses bounded SQLite metadata queries instead of scanning binaries at query time.
  • Dashboard uses a Dynamic-Island-aware centered welcome section with the VOXos logo, app purpose, compact theme controls, fixed taller bottom Persona/Voice actions, inline long-press voice capture with footer subtitles, and edge-to-edge main content.
  • Welcome controls include a user avatar entry point for account integrations. The sheet lists Google, Composio, Apple, Microsoft, GitHub, Dropbox, Notion, Slack, Calendar, Mail, Contacts, and Photos as configurable sources pending OAuth/provider credentials, with one inline provider detail expanded directly under the selected integration at a time.
  • Composio integration has its own dashboard grid app and account integration panel. VOXos persists the Composio API key, user ID, default toolkit, and API base URL in local SQLite, can test the connection, and loads connected account metadata for the configured user.
  • Hugging Face integration has its own dashboard grid app and account integration panel. VOXos stores the Hugging Face access token in iOS Keychain, persists non-secret model, chat endpoint, scope, namespace, repository, and repo type defaults in local SQLite, tests the OpenAI-compatible Inference Providers chat endpoint, refreshes Hub profile/repository/dataset/Space metadata, and syncs Hugging Face bucket list/file metadata directly on iPhone or iPad without downloading object contents. Each bucket metadata sync records local history with current totals, added/modified/deleted/unchanged counts, and recent file-level change rows.
  • Hugging Face setup uses direct access-token entry. The token field includes an inline VOXos-styled guide for creating a fine-grained Hugging Face token with only the inference, repository, and bucket read/list scopes the user wants VOXos to use.
  • CrabTalk integration has its own dashboard grid app. VOXos persists the remote daemon URL, default agent, transport note, enabled state, and host notes in local SQLite, stores the optional access token in iOS Keychain, can test common HTTP bridge endpoints for a reachable CrabTalk daemon, and opens a typed/voice chat panel after a successful daemon test.
  • Mac Link integration has its own dashboard grid app for a trusted macOS companion server. VOXos persists the Mac server URL, display name, bridge mode, notes, and enabled state in local SQLite, stores the optional pairing token through secret storage, tests LAN, VPN, or HTTPS tunnel reachability, can post approved commands to the Mac for direct companion actions or an allowlisted KDE Connect bridge, shows high-resolution live previews for all active Mac screens arranged by their real display coordinates with cursor capture where macOS permits it, has compact and full-screen preview zoom up to 500%, provides a full-screen mouse/keyboard switcher, and includes a phone touchpad for mouse movement, left/right click, double click, and scroll through the Mac server.
  • Mac Link now follows the active Google profile selected from the Profile/top navigation account sheet. iPhone and iPad devices on the same active Google account can refresh, list, share, and adopt shared Mac endpoints through a Supabase voxos_profile_maclink_endpoints table; switching Google accounts switches the visible Mac endpoint namespace. Local SQLite remains the device cache, not the shared source.
  • Mac Link now reads server version and capability flags from the macOS companion health endpoint, then shows which bridge features are ready or blocked by server flags such as clipboard read, terminal access, web terminal support, KDE Connect, and power actions.
  • Mac Link now exposes compact clipboard and Mac file-search controls. VOXos can request Mac clipboard reads, write text to the Mac clipboard, and send Mac Spotlight file-search commands through the trusted companion server.
  • Terminal integration has its own dashboard grid app backed by Mac Link. The macOS companion server exposes guarded tmux endpoints for status, session list/create, pane capture, input send, Terminal/iTerm window/tab/pane discovery, native target capture/send, opening a tmux session in Terminal or iTerm, and optional ttyx URL launch. VOXos lists tmux sessions alongside Terminal and iTerm targets so the user can switch in place and continue from the phone. Pilotty and Agent Browser reuse the same global Mac Link connection, show inherited connection state, preflight terminal readiness before sending commands, and refresh terminal output after automation commands run.
  • Gateway runtime is available in the backend for OpenClaw-style expansion without changing the native UI. It stores sessions/messages in SQLite, accepts generic and channel-specific ingress for VOXos iOS, WebChat, CLI, Telegram, Slack, Discord, WhatsApp-style payloads, and trusted Mac bridges, returns skill/tool-profile route proposals with confirmation requirements, registers trusted nodes with heartbeat support, exposes built-in VOXos skill metadata, tracks executable background job records, protects public adapters with optional channel signatures and rate limits, and can execute safe read adapters while keeping write/destructive routes staged.
  • VOXos now has an OpenClaw-inspired Gateway first slice. Existing native surfaces expose Gateway session queues, trusted nodes, skills/tool profiles, WebChat/channel planning, and runtime health as read-only rows in Daemons, Jobs, Device Fleet, Service Health, Browser Tasks, Memory Answer, Automation Gallery, and global search. The Mac Link server also exposes GET /api/gateway/status / GET /gateway/status with nodes, skills, channels, tool profiles, jobs, and capability flags without enabling automatic channel messages or destructive execution.
  • The iOS app can fetch that Gateway payload from Mac Link, render a Gateway status panel in Mac Link details, and update existing platform rows with live node, skill, channel, profile, and job readiness counts while keeping Gateway execution disabled.
  • Terminal refreshes now preserve the existing terminal output when a capture fails, skip publishing unchanged captures to reduce visible flicker during auto-refresh, and show the last capture freshness in the Terminal status line.
  • The Composio detail page now loads connected accounts automatically on open, shows active services as tappable rows, and opens a voice-first service panel. The user holds Voice, speaks a request, reviews the heard transcript plus the LLM-rephrased information/filter/action prompt VOXos will actually send, and approves that intent before anything goes to Composio. Gmail, Drive, and GitHub service requests are wired to safe read-only Composio tools; unmapped services show the next toolkit-mapping step.
  • Composio Gmail voice prompts are now service-aware, so requests such as filtering mails with attachments are rephrased into Gmail-specific read intents using Gmail message fields, labels, snippets, attachment indicators, and the GMAIL_FETCH_EMAILS action before approval.
  • Composio Gmail responses render as compact structured mail rows with sender, subject, snippet, date, and category. Follow-up prompts can filter the current visible mail result set for transactions, receipts, senders, brands, or keywords before another service call is made.
  • Composio Gmail rows expand inline on tap to show the detailed fields returned for that mail, so compact previews can open into full message context without leaving the service chat.
  • Composio now has a generic service-result renderer for non-Gmail connectors, including safe Google Drive and GitHub read mappings, expandable returned-field rows, follow-up filtering against the current service result set, setup cards for unmapped toolkits, and recent result history stored in local SQLite.
  • Composio GitHub service requests now use GITHUB_LIST_REPOSITORIES_FOR_THE_AUTHENTICATED_USER to list accessible repositories, show the repository count loaded into VOXos, and render repo rows with returned metadata for quick inspection. GitHub commit follow-ups route to GITHUB_LIST_COMMITS and use the current top visible repository when the user does not specify owner/repo.
  • Composio detail content auto-scrolls to the latest service result or conversation update so the user does not have to keep dragging to the bottom after each turn.
  • Google integration has a native OAuth foundation from the profile/account sheet, Persona screen, Settings, and voice commands such as can you access Gmail. It persists Google config/status locally, supports multiple connected Google accounts with one active account, and starts consent when OAuth client details are configured. The active Google account is also the profile boundary for Mac Link endpoint/settings sync.
  • After a Google account is connected, a parent Google grid item appears with the active account label. Tapping it opens a Google services sheet, and supported services such as Gmail, Drive, Photos, Calendar, and Contacts route into their own detail pages with Granted or Reconnect scope state.
  • Photos is now a standalone dashboard app backed by Google Photos OAuth scope. It checks connected Google accounts for Photos Library readonly access, offers connect/reconnect and service-sheet actions, fetches recent Google Photos media metadata through the Photos Library API, and shows account, filename, media type, dimensions, date, and description without downloading media binaries.
  • Google service detail pages show account scope state and placeholders for the next provider API layer, including Gmail metadata, Drive files/storage, Photos metadata, Calendar events, and Contacts/People context.
  • Schedule refresh aggregates every connected native Google account with Calendar readonly scope plus every Composio Google Calendar connected account for the configured VOXos user. VOXos fetches upcoming primary-calendar events, refreshes Google OAuth tokens when needed, stores event metadata and source provenance in local SQLite with account-specific IDs, replaces the previous schedule cache on each refresh so stale events from another account are cleared, renders a compact seven-day calendar strip plus date-grouped event rows, shows the provider/account directly under each event title, opens a scrollable event details sheet with upstream source title/link/provider payload on row tap, and shows concise VOXos alerts instead of raw provider JSON when an integration fails.
  • Schedule event details include an intelligence card that classifies common event types such as train tickets, flights, bookings, interviews, payments, deliveries, medical appointments, and generic meetings. It extracts useful fields like PNR, flight number, booking ID, amount, tracking number, meeting link, company, role, and source link from the event description or mail/provider payload, shows next steps and missing-field warnings, and optionally asks the configured local/cloud LLM for a compact meaning summary.
  • Travel has a refresh flow that scans connected native Gmail accounts and Composio Gmail accounts for travel-related messages over a user-selected mail date range. VOXos fetches candidate mail metadata for flights, trains, PNRs, hotels, bookings, reservations, tickets, itineraries, cabs, and trips, extracts structured category/provider/reference/date/location fields with missing-field warnings, and saves the metadata into local SQLite without storing full mailbox contents. Travel rows and detail sheets show source provenance from Gmail or Composio, account, sender, subject, message ID, compact raw metadata, and warnings such as missing PNR, travel date, or route.
  • Receipts is now a receipt scanner and expense manager. Users can add bills manually, scan with Camera, or import a bill image from Photos. VOXos normalizes the image orientation, runs full-page plus full-width and two-column tiled Vision OCR to prefill merchant, actual receipt total, INR-first currency, category, and spending/borrowed/lent/reimbursable type, then opens a VOXos-themed review sheet where the user can inspect the photo, edit detected fields, adjust raw OCR text, save, or discard before anything is persisted. Scan review detects likely duplicates by normalized OCR hash or merchant/date/amount and requires an explicit Save Anyway selection before storing another copy. Confirmed receipts save the original scan image plus raw OCR text with metadata in local SQLite, show spend/borrowed/lent totals in INR by default, category summaries, editable rows, photo preview, OCR text review, and appear in global search. The receipt editor uses VOXos-themed sheet chrome and controls. Amount extraction handles Indian formats such as 1100/-, Rs 1,100, and ₹1100.00 while ignoring small item counters such as Total items 3.
  • Receipts can generate Persona suggestions without auto-writing profile data. VOXos analyzes saved receipts for frequent merchants, top spend categories, reimbursement context, borrowing/lending patterns, and useful receipt notes, then opens the Persona review queue so the user can approve or discard each proposed update.

  • Dashboard scrolling hides the visual scroll indicator, and the icon grid uses increased top/bottom padding with larger icon targets relative to the pin and info controls.
  • Dashboard sub-application labels are single-line and truncate with an ellipsis inside the icon width so long names like Notifications do not stretch the grid.
  • Dashboard grid uses a fixed compact icon layout with flat app icons. Grid labels stay single-line and truncate inside the icon width; secondary account text is hidden from the grid so items such as Google only show the app title. Long-pressing a grid icon opens the app information popover with purpose, use cases, and configuration details.
  • Every dashboard sub-application icon has vertically stacked pin and info controls with matching dimensions.
  • Dashboard app icons are ordered by recent use first. Grid taps slide the dashboard left and open a separate detail page from the right while keeping a narrow parent-page strip visible; the detail page repeats the compact theme, dark mode, info, and account navigation controls at the same top level as the main page, includes a compact per-app description/setup section, supports a right swipe back to the main page, and pinning an open item closes the detail page and leaves the card persisted in the pinned section below.
  • Detail-page intro sections use neutral grey informational typography for descriptions and setup/use-case copy, keeping accent colors reserved for app icons, actions, and state indicators.
  • iPad dashboard grid scrolling skips the date-wheel-style sound/haptic offset observer for smoother scrolling. iPhone keeps rate-limited scroll tick feedback.
  • Persona is a dedicated full-screen area launched from the footer or voice command instead of a dashboard sub-application card. It accepts typed details, voice entries, or uploaded files, extracts local persona facts such as current role, current company, current work, location, skills, interests, and education, and stores them in SQLite with source and confidence.
  • Persona screen shows compact source counters, detailed fact rows, delete/clear controls, and profile areas for work, life, health, taste, goals, and approved data sources.
  • Persona facts can be deleted individually or cleared as a group, and matching attribute/value pairs replace earlier copies to avoid repeated duplicate facts.
  • Call card provides an in-app status check flow: ring, answer, capture spoken status, and save the result to local SQLite.
  • Call card can poll a configurable remote value every 5 seconds while the Call section is open; when the remote value is 0, VOXos opens the Call card and keeps ringing every 5 seconds until a later poll reads 1.
  • Alarm card provides a compact clock lab with persisted local alarms, one-minute alarm time adjustment, iOS notification scheduling, timer countdown controls including seconds, smaller adaptive ring text for long durations, stopwatch start/pause/reset with millisecond display, and lap capture while VOXos is open.
  • Music card is now a real compact player backed by bundled copyright-safe MP3 demo tracks, local MP3/audio folder scanning through the iOS Files picker, and direct audio URL playback. It supports play/pause, previous/next, seek, volume up/down, mute, current progress, a URL input, embedded provider playback for Spotify/YouTube/JioSaavn links, persisted local track bookmarks, background audio mode, lock-screen/Home Screen Now Playing controls, and tappable local/demo playlists while matching the VOXos rectangular UI.
  • Documents is a native grid micro-app for curated IDs, invoices, warranties, forms, policies, and contracts. It imports PDFs, images, and text files through the iOS document picker, extracts local PDF text and Vision OCR for images, stores metadata/extracted text in SQLite schema version 10, supports manual metadata entry, local filtering, delete, expiring-soon search provenance, and voice commands for open/import/search/summarize/expiry checks.
  • Approvals is a native grid micro-app that aggregates pending confirmation state across VOXos. It shows risky command confirmations, ask-to-edit proposals, receipt scan reviews, Persona update reviews, and automation dry-run setup prompts in one surface with approve/cancel/open-source actions, voice commands for checking or approving the next item, Daily Briefing priority rows, and global search provenance for pending approval state.
  • Inventory is a native Household Inventory grid micro-app for pantry, groceries, appliances, warranties, receipt references, expiry dates, and household notes. It stores local metadata in SQLite schema version 11, supports manual add/delete, quantity/unit/location/category fields, expiry/warranty attention states, local filtering, global search provenance, and voice commands for open/search/summarize/expiry checks.
  • Relationships is a native Relationship CRM grid micro-app for people context, promises, notes, companies, tags, source provenance, and follow-up dates. It stores local metadata in SQLite schema version 12, supports manual add/delete, follow-up attention states, local filtering, global search provenance, and voice commands for open/search/summarize/follow-up checks.
  • Projects is a native Developer Projects grid micro-app for local repos, branches, remotes, terminal commands, project status, and build/test notes. It stores local metadata in SQLite schema version 13, supports manual add/delete, Terminal and Mac Link jump actions, local filtering, global search provenance, and voice commands for open/search/summarize/status checks.
  • Books is a native Book Library grid micro-app for books to read, current reading, notes, quotes, formats, tags, and source context. It stores local metadata in SQLite schema version 14, supports manual add/delete, current-page progress, local filtering, global search provenance, and voice commands for open/search/summarize/current-reading checks.
  • Assets is a native Asset and Warranty Vault grid micro-app for devices, serial numbers, purchase dates, warranty dates, receipt references, document references, values, and repair notes. It stores local metadata in SQLite schema version 15, supports manual add/delete, warranty attention states, local filtering, global search provenance, and voice commands for open/search/summarize/warranty checks.
  • Deliveries is a native Deliveries and Orders grid micro-app for order confirmations, merchants, carriers, tracking numbers, delivery status, ETA dates, return windows, OTP hints, refunds, and package notes. It stores local metadata in SQLite schema version 16, supports manual add/delete, attention states for near ETAs/returns, local filtering, global search provenance, and voice commands for open/search/summarize/status checks.
  • Snippets is a native Clipboard and Snippets grid micro-app for trusted reusable text, prompts, terminal commands, templates, addresses, and clipboard handoff notes. It stores local metadata in SQLite schema version 17, supports manual add/delete, sensitive snippet labeling, local filtering, global search provenance that hides sensitive snippet body text, and voice commands for open/search/summarize checks.

The Sounds dashboard app controls tap/selection tones, voice press tone, result tones, notification tones, status call ringtone, master sound/haptic toggles, haptic strengths, preview actions, and quiet/balanced/silent profiles.
  • Sound and haptic preferences persist in local SQLite and are applied to shared UI feedback, foreground VOXos notification banners, footer voice press feedback, status calls, and remote call trigger ringing.
  • Settings subsections use light horizontal separators and body-sized labels for Appearance, Voice, Cloud, Local Models, and Storage groups.
  • Settings now starts with a compact System Health subsection that summarizes readiness for voice input, local LLM, Google, Composio, Hugging Face, Mac Link, Files, Music, and Widgets so setup gaps are visible before manual or automated testing.
  • Settings includes a remote call trigger URL, an on/off control to stop polling and ringing, last fetched value, status, and manual check action. The trigger defaults off, and the trigger URL, enabled state, and last value are stored in local SQLite.
  • Settings includes a Widgets section for Lock Screen/Home widget snapshot sharing, selected VOXos app shortcuts, and Live Activities. The widget config persists in SQLite, writes a compact shared snapshot through the group.com.shyamzzp.voxos App Group, and exposes manual controls to publish a snapshot or start/stop Indexing and Call Trigger Live Activities. Widget health shows App Group read/write status, last publish/read time, included widget apps, feature count, and concise recovery text when the App Group or snapshot is unavailable.
  • The dashboard can show a non-blocking offline/provider-degraded banner when network-like provider failures are detected. It explains what still works locally, including Notes, Persona, Receipts, Files, Local search, cached Schedule and Travel rows, and the selected local model, with a direct path back to Providers for setup or retry.
  • Providers includes a SQLite-backed background sync queue foundation. VOXos can queue deferred provider-health refreshes and retry supported work types such as Composio account loading, Google Photos refresh, Terminal bridge refresh, file reindexing, and travel mail scans from one compact queue panel.
  • Providers can generate a privacy-safe diagnostics export as pretty JSON. The export includes app settings summary, provider readiness, recent command audit summaries without full transcripts, local database/migration/widget health, Keychain presence labels, and redacted error summaries; it is copied locally and can be shared from the VOXos-styled Providers panel.
  • VOXos persists a small rolling diagnostic breadcrumb buffer for route changes, command intents, provider refresh results, and redacted error context. Breadcrumbs are included in diagnostics export and the Debug Data Inspector without raw transcripts, tokens, provider payloads, or secret values.
  • Debug builds include a Local Data Inspector dashboard app. It shows SQLite table counts, redacted latest-row summaries, schema migration status, Keychain presence flags, and App Group widget snapshot health without exposing raw tokens, card numbers, CVV, image blobs, OCR text, message bodies, notes, or provider payloads.
  • Feature planning includes a checklist generator at scripts/generate_feature_verification_checklist.py. It reads a GSD phase plan and creates a markdown verification checklist covering build/install, primary interaction, error state, persistence, regression checks, and applicable automated tests.
  • VOXos supports stable internal deep links for dashboard apps and key Settings subsections. Examples include voxos://feature/files, voxos://feature/composio, voxos://feature/terminal, voxos://settings/appearance, voxos://settings/voice, voxos://settings/local-models, voxos://settings/storage, and voxos://feature/settings?section=google. Invalid VOXos links fail with a concise in-app message instead of silently doing nothing.
  • Rules is a dashboard app for local-first automation. It persists rule title, trigger, action, target, enabled state, last run, next run, and failure reason in SQLite, runs due rules while VOXos is active, and supports safe actions such as opening apps, refreshing providers, ringing status calls, creating notes, staging Persona proposals, and surfacing approval requirements for risky Mac/provider actions. Rules also include a dry-run simulator: the user enters sample trigger data, previews the proposed action, target, risk, confirmation need, and safety note, and disabled rules must be dry-run before they can be enabled. Dry runs do not mutate local data, navigate, ring calls, or call providers.
  • VOXos now ships a WidgetKit extension with VOXos Status and VOXos Apps widgets. The Apps widget shows user-selected VOXos applications, each with a compact status line and a deep link into the matching VOXos detail page. The extension also includes an ActivityKit Live Activity/Dynamic Island configuration for active indexing and call-trigger status.
  • Dark/light mode, theme selection, accent color, font family, and display scale settings saved in local SQLite.
  • Full-screen app presentation with status bar hidden.
  • Wi-Fi install helper for the paired iPhone via scripts/install_wifi_iphone.sh.
  • Location/weather context using CoreLocation and weather APIs. The app requests a fresh device fix, asks for temporary precise location when iOS is in reduced-accuracy mode, and no longer uses a hardcoded simulator fallback.
  • Local context analytics now ask the configured on-device GGUF LLM to explain the meaning of the current place, weather, rain, wind, and location-confidence signals instead of restating raw values. If no local model is ready, VOXOS uses a compact deterministic fallback explanation.
  • Foreground wake phrase and bundled App Shortcuts. Wake word settings now arm the feature without immediately starting the listener; the explicit wake listener control warns about microphone/speech access and listens for VOXos or Hey VOXos. VOXos also ships App Intents/App Shortcuts for opening Voice and routing directly to selected apps such as Composio, Persona, Files, Calls, Wallet, and Settings without the user manually creating Shortcuts.
  • Inline voice capture uses the custom VOXos alert overlay, not native iOS alerts, when the microphone does not start, no speech is detected, or local/cloud command processing fails. The overlay follows the app design system and includes next-step guidance for permissions, local LLM setup, Cerebras, or backend connectivity.
  • Voice is available as a normal dashboard grid app with a compact chat panel. Footer Voice long-press stays focused on capture-and-send, while spoken and typed messages both run through the same command processor so phrases like start Google login can trigger account integration setup.
  • Voice now acts as the first command inbox foundation: it shows interpreted intent, pending approvals, resumable confirm/cancel controls, and an action receipt timeline for completed, failed, cancelled, and setup-needed commands. Briefing and Search expose quick conversational follow-ups that route back through Voice instead of forcing a full manual navigation path.
  • Voice includes an approval-first Daily standup panel for plan, blockers, focus, travel, schedule, and reminders. Approved standups are saved locally to Notes and Status Checks with source Daily standup, then written as an action receipt.
  • Daily Briefing is conversational at the item level. Each briefing row can ask why it matters, what to do first, what changed, or request a reminder suggestion; the selected item context is routed back through Voice.
  • The dashboard shows ambient status whispers for active long-running work such as Files indexing, Music scanning, model downloads, Composio/Schedule/Travel/Photos refreshes, Terminal capture, Mac preview, and sync queue work. Each row is non-blocking and opens the relevant detail page.
  • Voice supports Ask-to-edit confirmation cards for small mutations. Users can request edits such as change receipt category to travel or set Mac refresh to 2 seconds; VOXos shows target, field, before, and after values, then saves only after approval.
  • Natural-language settings changes reuse the same approval card for commands such as set theme to graphite, switch to dark mode, set voice engine to cloud, or use Apple Speech. The card shows the interpreted Settings path and old/new values before saving locally.
  • Search supports conversational refinement. Follow-ups such as only receipts, exclude stale files, fresh results, and last month apply persistent filters, show compact chips, and can be cleared without losing the current query.
  • Messages now includes a local-only outbound composer. Users can save unsent drafts with recipient, channel, subject, source context, and multiline body, or say commands such as draft a message to Alex saying I will be late; VOXos stores the draft in local SQLite, adds it to Search with unsent-draft provenance, and records an action receipt without sending anything.

Storage Direction​

  • Local-first data lives in Application Support/VOXOS/voxos.sqlite.
  • SQLite migration backups live under Application Support/VOXOS/Backups/ and are created before versioned schema upgrades on existing installs.
  • Supabase migration should preserve table boundaries where possible.
  • Large generated artifacts should live outside SQLite under Application Support/VOXOS/IndexCache/.
  • Local model files live under Application Support/VOXOS/Models/; model metadata lives in SQLite.
  • Native GGUF inference is bundled through voice_xcode/Frameworks/llama.xcframework. Core ML model storage is present, but generation still needs a model-specific adapter.
  • Widget snapshots are encoded into App Group user defaults under group.com.shyamzzp.voxos; the widget extension reads only that compact snapshot and does not access SQLite directly.
  • Mac Link development server lives at scripts/voxos_mac_link_server.py; setup notes live in docs/MAC_LINK_INTEGRATION.md.
  • Terminal bridge setup notes live in docs/TERMINAL_INTEGRATION.md; it requires Mac Link plus VOXOS_MAC_LINK_ALLOW_TERMINAL=1 and tmux on the Mac.
  • NDA draft for S&S Services and Consultants lives at docs/NDA_S_AND_S_SERVICES_AND_CONSULTANTS.md.
  • The bundled SmolLM2 135M GGUF local LLM lives in voice_xcode/VOXOS/BundledLocalModels/; Xcode packages it as an app resource and VOXOS registers the read-only bundle path in SQLite at startup.
  • The bundled Parakeet v3 Core ML source lives in voice_xcode/VOXOS/BundledVoiceModels/; Xcode packages it as an app resource, and VOXos builds the FluidAudio AsrModels object directly from those packaged Core ML files.

Location Notes​

  • On a physical iPhone, VOXOS uses the device CoreLocation fix after location permission is granted.
  • On Simulator, iOS does not automatically use the Mac's real location. Set a simulator location from the Simulator location menu or Xcode scheme options before refreshing weather.
  • If no simulator location is configured, the app shows a timeout instead of showing incorrect fallback weather.
